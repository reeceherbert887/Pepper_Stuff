# Pepper Freshers Personality Test Setup Guide

## Overview

The goal of this project was to create an interactive Freshers' Fair activity using Pepper.

Instead of Pepper just sitting there as a showcase robot, I wanted people to actually interact with it and hopefully get interested in robotics and the society.

The idea was to create a robotics personality test that runs on Pepper's tablet.

Students answer a few questions and Pepper tells them what type of roboticist they are.

Examples include:

* AI Researcher
* Rescue Robotics Engineer
* Embedded Systems Engineer
* Autonomous Systems Engineer
* Computer Vision Specialist
* Drone Specialist

At the end Pepper speaks the result and the user is shown a QR code to join the society.

---

# Project Structure

The project is split into two parts:

## Tablet Interface

Built using:

* HTML
* CSS
* JavaScript

Files:

```text
index.html
style.css
script.js
```

The tablet handles:

* Displaying questions
* Collecting answers
* Calculating the result
* Displaying the final outcome

---

## Pepper Side

Built using:

* Choregraphe
* Python
* ALMemory
* ALTextToSpeech

Pepper handles:

* Opening the tablet application
* Receiving the quiz result
* Speaking the result aloud

# Development Process

## Stage 1 - Basic UI

Created a simple HTML page containing:

* Start screen
* Quiz screen
* Result screen

Initially tested locally using:

```bash
python3 -m http.server 8000
```

and then opening:

```text
http://localhost:8000
```

This allowed the UI to be developed without needing access to Pepper.

---

## Stage 2 - Quiz Logic

JavaScript was used to:

* Store questions
* Store answer types
* Track scores
* Calculate the winning category

Once complete the result screen displayed:

```text
You are an AI Researcher!
```

or whichever category scored highest.

---

## Stage 3 - Pepper Tablet Testing

The project was imported into Choregraphe and launched using a Show App box.

The tablet UI loaded correctly.

However Pepper initially could not react to the result because the tablet and robot were completely separate systems.

The tablet knew the result.

Pepper did not.

---

## Stage 4 - Speech Testing

A Python Script box was created.

Initially it was tested using:

```python
tts.say("Hello from Python")
```

This confirmed:

* Python box worked
* ALTextToSpeech worked
* Pepper could speak

---

## Stage 5 - Connecting Tablet To Pepper

QiMessaging was added to the HTML project.

JavaScript raises:

```text
PepperFreshers/Result
```

using ALMemory.

The Python Script box subscribes to:

```text
PepperFreshers/Result
```

and waits for incoming data.

When a result arrives:

```python
self.tts.say(str(value))
```

is executed.

# Final Working Flow

```text
Start

Result calculated
↓
Result displayed
↓

↓
Python listener receives event
↓

```

# Notes

One of the biggest lessons from this project was that Pepper's tablet and Pepper itself are effectively separate systems.

Getting the UI working was relatively straightforward.

The difficult part was getting information from the tablet application back into Choregraphe so Pepper could actually react to what the user selected.

This was solved using ALMemory events and a Python listener box.

Once that bridge was working, Pepper could respond dynamically to any result generated by the quiz.
